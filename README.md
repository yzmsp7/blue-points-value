# Blue Points 價值查詢

在 [awardsworldwide 商城](https://ibm65984twn.awardsworldwide.com/) 的每張商品卡最下面插一行，
抓**藍點數**，一鍵背景查該商品的**真實市價**（附一鍵開商品連結），算出 **每點值多少錢**，
幫你判斷哪些商品兌換最划算。

## 安裝

1. Chrome 開 `chrome://extensions`
2. 右上角開啟 **開發者模式**
3. 點 **載入未封裝項目**，選這個資料夾（含 `manifest.json` 的那層）
4. 之後改了程式碼，回這頁按該擴充的 **↻ 重新載入**

## 使用

1. 登入商城，開任一商品列表頁（例：electronics catalog）
2. 每張商品卡最下面多一行：`X 點 · [市價欄] · [查價] · 每點價值`
3. 點 **查價** → 背景查搜尋引擎（不開分頁）**自動抓第一筆價格**填回市價欄
4. 每點價值即時算出；數字越大代表這件商品換得越划算。抓到的價格出現 **⚠** 表示每點價值超出合理範圍（NT$1~15/點），很可能抓錯商品，請自行覆寫市價欄
5. 右下角浮動控制：**幣別 TWD ⇄** 切換 TWD／USD；查到的價格是 TWD，切 USD 時用**自動抓的即期匯率**換算（唯讀顯示，不用手設）
6. 你填/抓到的市價、幣別、匯率都會記住，切幣別/換頁/重開都不跑掉
7. 換頁或商品卡動態載入時會自動補上（MutationObserver）

## 每點價值怎麼算

```
每點價值 = 真實市價 / 藍點數
（市價一律以 TWD 存 chrome.storage；USD 模式用匯率換算顯示）
```

不用商城自帶的 `gtag_view_item_price`——它只是「點數 × 固定匯率」的美元標示，與真實市價無關。

## 自動抓價怎麼運作

- 「查價」→ `background.js`（service worker）背景 `fetch` 搜尋引擎「商品名 價格」。
- **備援鏈：Bing → DuckDuckGo →（保底）Google**。第一個抓到合理價的就贏、命中即停。
  Bing/DDG 不需登入 cookie、限制較寬鬆；Google 因常被行為性限流，只當最後手段（帶你已登入的 cookie 繞 CAPTCHA）。
- 抓 HTML 第一個合理價格 → 寫入 `chrome.storage.local` → 商城分頁的 `onChanged` 即時填入該卡。
- **全程不開任何分頁。** 批次「查本頁未查的」每筆間隔 1.5s，避免連打被限流。
- 三來源都查不到會顯示失敗訊息（被限流／逾時／查不到），手動填市價欄即可。

## 疑難排解

- **沒出現浮層**：確認你在 `ecommerce/*` 頁面、且擴充已載入並重新整理頁面。
- **某商品顯示「點數?」**：代表沒抓到藍點數（點數文字沒有「點/points」標籤，或位置太遠）。
  把那張商品卡顯示點數的那段 HTML 貼出來，就能修正抓取規則。

## 檔案

| 檔案 | 作用 |
|------|------|
| `manifest.json` | MV3 設定：注入頁面範圍、腳本、storage 權限、背景 worker、圖示 |
| `content.js` | 商城頁：抓商品、插入徽章、價值計算、每點價值 ⚠ 檢查、幣別切換、批次查價 |
| `background.js` | 背景 fetch 搜尋引擎備援鏈（Bing→DDG→Google）抓市價寫回 storage（不開分頁） |
| `overlay.css` | 徽章與浮動控制樣式 |
| `128.png` | 擴充圖示 |
